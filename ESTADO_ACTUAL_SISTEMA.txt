2. ARQUITECTURA DEL SISTEMA
================================================================================

DIMENSIONES ACTUALES:

Estados (X ∈ ℝ¹⁵):
  • Dual quaternion (dq)      [8 elementos] - Pose (rotación + posición)
  • Velocidad angular (w_b)   [3 elementos] - Velocidad angular en cuerpo
  • Velocidad lineal (v_b)    [3 elementos] - Velocidad lineal en cuerpo
  • Arc length (s)            [1 elemento]  - Parametrización de trayectoria

Controles (u ∈ ℝ⁵):
  • Fuerza propulsora (F)     [1 elemento]  - Control vertical
  • Torques (τ₁, τ₂, τ₃)     [3 elementos] - Controles de orientación
  • Arc velocity (u_s)        [1 elemento]  - Control de progreso en trayectoria

Parámetros (p ∈ ℝ¹⁸):
  • Referencia dual quat (dq_d)     [8 elementos]
  • Referencia velocidad angular     [3 elementos]
  • Referencia velocidad lineal      [3 elementos]
  • Controles nominales             [4 elementos]

DINÁMICA:

  d/dt[dq]    = f(dq, u)              [Dinámicas Lie en SE(3)]
  d/dt[w_b]   = τ / J                 [Ecuación de momento angular]
  d/dt[v_b]   = F/m - g·e_z           [Ecuación de movimiento lineal]
  d/dt[s]     = u_s                   [Parametrización de arco]

COSTO (FUNÇÃO OBJETIVO):

  J = w₁·ln_error²   [Lie-invariant error]
    + w₂·control²    [Control effort]
    + w₃·u_s²        [Arc velocity smoothness]
    + w₄·contour²    [Lag-contour decomposition]
    - w₅·progress    [Encourage trajectory progression]

3. FLUJO DEL ALGORITMO MPC
================================================================================

Cada iteración (100 Hz = 10 ms):

 1. RECIBIR ESTADO ACTUAL
    └─ x_k ∈ ℝ¹⁵ (medido o estimado del simulador)

 2. PROCESAR ARC LENGTH DINÁMICO
    └─ Calcular s_current += u_s_optimal * dt (de iteración anterior)
    └─ s_current = clip(s_current, 0, arc_length_total)

 3. RECONSTRUIR REFERENCIAS DINÁMICAS (NEW)
    └─ Para j = 0 hasta N_prediction:
       ├─ s_pred_j = s_current + j * dt * s_nominal_rate
       ├─ pos_ref_j = position_by_arc_length(s_pred_j)
       ├─ quat_ref_j = quaternion_by_arc_length(s_pred_j)
       ├─ vel_ref_j = velocity_by_arc_length(s_pred_j)
       └─ Construir X_d_j y parámetros p_j

 4. RESOLVER OCP CON ACADOS
    └─ minimize J(u, X_d) sujeto a:
       ├─ x̄ = f(x, u)              [Dinámicas]
       ├─ 0 ≤ F ≤ F_max            [Límites fuerza]
       ├─ -τ_max ≤ τ_i ≤ τ_max     [Límites torques]
       ├─ 0.01 ≤ u_s ≤ 3.0         [Límites arc velocity]
       └─ ||q|| = 1                [Restricción cuaternión]

 5. EXTRAER CONTROL ÓPTIMO
    └─ u_optimal = [F*, τ*, u_s*]

 6. APLICAR CONTROL A SIMULACIÓN
    └─ Integrar dinámicas con RK4 un paso: x_{k+1} = x_k + f(x_k, u_optimal) * dt

 7. GUARDAR DATOS Y AVANZAR ARCO
    └─ Guardar X, u, costos
    └─ s_current += u_s_optimal * dt

 8. REPETIR HASTA t_final = 30 s

4. INNOVACIÓN CLAVE: DESACOPLAMIENTO TIEMPO-ARCO
================================================================================

PROBLEMA ORIGINAL:
  • Parametrización de trayectoria en arco (s): suave, geométrica
  • Control en tiempo (t): fijo, no adaptable
  • Implicación: Velocidad de progreso sobre trayectoria fija
  • Limitación: No se puede optimizar el "ritmo" de seguimiento

SOLUCIÓN IMPLEMENTADA:
  • Añadir s como variable de estado dinámica
  • Añadir u_s como variable de control independiente
  • MPC puede ahora elegir velocidad óptima sobre trayectoria
  • Referencias reconstruidas dinámicamente: s → t(s) → referencias

BENEFICIOS:
  ✓ Mayor flexibilidad en el control
  ✓ Mejor eficiencia energética posible
  ✓ Control de "marcha" adaptable
  ✓ Decoupling natural de geometría (arco) y cinemática (tiempo)

8. PARÁMETROS DE SINTONIZACIÓN ACTUAL
================================================================================

En nmpc_acados.py (líneas 46-52):

  # Arc length rate control
  u_s_max = 3.0        # Máxima velocidad de progreso [m/s]
  u_s_min = 0.01       # Mínima velocidad de progreso [m/s]
  R_s = 0.1            # Ganancia de costo para u_s

En MPC2.py (línea ~510):

  s_nominal_rate = 0.5 # Velocidad nominal para predicción [m/s]

SIGNIFICADO:
  • u_s puede variar entre 0.01 y 3.0 m/s (ej: 1.2 m/s = 120% de nominal)
  • La predicción asume velocidad nominal de 0.5 m/s
  • R_s=0.1 penaliza poco la variabilidad en u_s (permite flexibilidad)

RECOMENDACIONES:
  • Monitorear u_s en el gráfico 5_Control_Actions.pdf
  • Si u_s está saturado (siempre 3.0 o siempre 0.01), aumentar R_s
  • Si u_s es muy variable, reducir R_s
  • Ajustar s_nominal_rate según velocidad deseada de la trayectoria
